# Code of Conduct

Animus Datalab is an engineering-focused open-source project. The community should be useful, precise, respectful, and safe for maintainers, users, and contributors.

## Expected behavior

- Be respectful and constructive.
- Assume good intent, but be precise when discussing technical risk.
- Keep discussions focused on reproducibility, auditability, security, architecture, operations, and maintainability.
- Use evidence when making claims about behavior, performance, security, or compatibility.
- Distinguish implemented behavior from roadmap or target-only behavior.
- Respect maintainers' time by opening focused issues and reproducible reports.

## Unacceptable behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing private information without permission.
- Publicly disclosing security vulnerabilities before coordinated disclosure.
- Spam, artificial promotion, or low-quality promotional activity.
- Misrepresenting roadmap, security posture, or implementation status.
