# ADR-0015: Модель Vault‑подобной аутентификации и гарантии редактирования

**Статус:** Принято

**Дата:** 2026-02-04

## Контекст
Секреты должны быть доступны только в DP, а CP не должен получать значения. Требуется модель аутентификации к Vault‑подобному хранилищу через Kubernetes и формальные гарантии redaction на всех путях.

## Решение
- DP использует Kubernetes ServiceAccount JWT для аутентификации в Vault‑подобном backend и получает краткоживущий токен.
- Доступ к значениям секретов выполняется только в DP во время исполнения; CP хранит лишь ссылки (`secret_ref`).
- Redaction применяется к логам, ошибкам, аудиту и артефактам; утечка значений запрещена на всех путях.

## Обоснование
- Короткоживущие токены минимизируют риск компрометации.
- Разделение CP/DP соответствует принципу минимальных привилегий и существующим ADR.
- Единый редактирующий слой упрощает формальное тестирование на отсутствие утечек.

## Последствия
- DP зависит от доступности Vault‑подобного backend; ошибки отражаются как контролируемые, без раскрытия значений.
- Для новых провайдеров требуется реализовать тот же контракт redaction и TTL.
