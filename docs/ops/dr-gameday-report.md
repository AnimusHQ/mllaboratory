# Отчёт DR game‑day

## Контекст

- Цель: подтвердить восстановление платформы в пределах целевых RPO/RTO.
- Контур: тестовый (staging), без пользовательских секретов.
- Дата: 2026‑02‑05.

## Проведённые шаги

- Остановлены CP/DP.
- Восстановлены Postgres и MinIO/S3 из бэкапа.
- Запущены CP/DP и проверены `/readyz`.
- Выполнены базовые сценарии: чтение артефактов, создание/аппрув/экспорт модели.

## Результаты

- Фактический RPO: 30 минут.
- Фактический RTO: 45 минут.
- Потеря данных не обнаружена, кроме ожидаемой в рамках RPO.

## Частичные восстановления

- Восстановление только Postgres: сервисы запускаются, доступ к объектам отсутствует, ошибки корректные.
- Восстановление только MinIO: API отвечает `not_found` на метаданные, паник и повреждений состояния нет.

## Устаревший бэкап

- Бэкап, отстающий на 1 релиз, восстановлен с успешным прогоном миграций.
- Проверка совместимости API прошла без критических расхождений.

## Замечания и действия

- Актуализировать частоту бэкапов при повышении критичности RPO.
- Подготовить автоматизированный smoke‑набор для пост‑restore проверки.
