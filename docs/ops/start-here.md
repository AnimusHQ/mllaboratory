# Стартовый маршрут работы в Animus Datalab

**Версия документа:** 1.0

## Назначение
Документ описывает минимальный, проверяемый маршрут работы в платформе без скрытых шагов, что снижает риск неполного или неверного запуска. Он предназначен для первого запуска и подготовки доказательной базы (аудит, артефакты, экспорт).

## Предусловия
- Доступ к шлюзу (Gateway) и консоли.
- Действующая учётная запись.
- Для создания проекта требуются права администратора (роль `admin` или `platform_admin`).

## Шаги
1. **Вход и выбор проекта**
   - Войдите через шлюз (OIDC/SSO) и откройте консоль.
   - Если активный проект не выбран, консоль направит на экран выбора проекта.
   - Если проекта нет, создайте его (только администратор). После создания проект становится активным.

2. **Dataset и DatasetVersion**
   - Зарегистрируйте Dataset в проекте.
   - Создайте DatasetVersion и загрузите данные.
   - Убедитесь, что версия данных неизменяема и готова для Run.

3. **Run или PipelineRun**
   - Создайте Run (или PipelineRun) с явными входами:
     - `DatasetVersion`
     - `CodeRef` (commit SHA)
     - `EnvironmentLock`
     - `PolicySnapshot`
   - Дождитесь терминального состояния (succeeded/failed/canceled).

4. **Artifacts и доказательства**
   - Проверьте артефакты Run: контрольные суммы и доступ.
   - Сформируйте reproducibility bundle (если требуется регуляторная среда).

5. **Model Registry**
   - Создайте ModelVersion и привяжите provenance к Run и артефактам.
   - Проведите переходы статусов (validate → approve → deprecate при необходимости).

6. **Export**
   - Запросите экспорт версии модели (deny‑by‑default, требуется явное разрешение).
   - Убедитесь в наличии аудита операции экспорта.

7. **Audit Evidence**
   - Проверьте, что все ключевые действия отражены в `AuditEvent`.
   - При необходимости выгрузите аудит в SIEM (webhook/syslog), убедитесь в корректной доставке и отсутствии секретов.

## Проверка корректности
- Все create‑операции прошли идемпотентно (повтор не создаёт дубликаты).
- Run имеет зафиксированные входы (`DatasetVersion`, `CodeRef`, `EnvironmentLock`, `PolicySnapshot`).
- Артефакты доступны и проверяемы.
- Export выполнен только для `approved` версии.
- Audit append‑only и экспортируемый.
