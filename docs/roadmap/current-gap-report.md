# Текущий разрыв по отношению к roadmap.json (v3)

**Дата анализа:** 2026-02-02

## 1. Краткое резюме
Платформа соответствует базовым требованиям M0–M1 (разделение CP/DP, базовые доменные сущности, аудит и артефакты), но отсутствуют критические элементы для production‑grade: полноценный Data Plane, сквозное планирование/очереди/квоты, строгая привязка воспроизводимости и политики, а также инфраструктурные требования M5–M9 (SSO/RBAC, секреты, экспорт аудита в SIEM, наблюдаемость, HA/DR, упаковка и E2E‑приёмка). M2 находится в прогрессе, однако ключевые сущности и контракты воспроизводимости (CodeRef, EnvironmentDefinition/Lock, PolicySnapshot, параметры) не закреплены в схеме/репозиториях, а репродуцируемый экспорт входов Run отсутствует.

## 2. Соответствие текущим вехам
- **M0–M1:** реализованы основные доменные сущности, иммутабельность DatasetVersion, аудит (append‑only) и артефакты с проектной областью.
- **M2:** частично реализованы RunSpec, детерминированный план и dry‑run; отсутствуют:
  - нормализованные сущности **CodeRef**, **EnvironmentDefinition/Lock**, **PolicySnapshot**;
  - обязательные **параметры Run** и политика в RunSpec;
  - единый API для **reproducibility bundle** (экспорт входов Run);
  - контрактное описание RunSpec в OpenAPI.

## 3. Ключевые блокеры production‑grade (из roadmap.json)
1. **M3:** отсутствует Data Plane runtime executor (Kubernetes) и CP↔DP протокол.
2. **M4:** нет планировщика очередей/квот/ретраев/отмен.
3. **M5:** отсутствуют SSO (OIDC), RBAC‑матрица на всех эндпоинтах, управление секретами с TTL и SIEM‑экспорт аудита.
4. **M9:** отсутствуют наблюдаемость (Prometheus/OTel), HA/DR практики, SBOM/безопасность цепочки поставки, Helm‑упаковка с тестами обновлений, автоматизированная E2E‑приёмка.

## 4. Конкретные разрывы в репозитории
- **Контракты API:** OpenAPI не отражает RunSpec‑эндпоинты (`/projects/{project_id}/runs`) и будущий export входов Run.
- **Схемы БД:** отсутствуют таблицы для **CodeRef**, **EnvironmentDefinition**, **EnvironmentLock**, **PolicySnapshot**.
- **Воспроизводимость:** RunSpec не фиксирует параметры и snapshot политики; нет «одного клика» экспорта входов Run.
- **Политики и RBAC:** формальная матрица ролей существует в документации, но сквозная enforcement‑проверка по всем эндпоинтам отсутствует.
- **Helm‑миграции:** набор миграций в Helm‑chart покрывает только ранние версии и не соответствует полному дереву `closed/migrations`.

## 5. Конфликты/неоднозначности
- В `closed/meta/CODEX_INSTRUCTIONS.md` указано следование `roadmap.pilot.json`, тогда как **roadmap.json (v3) объявлен единственным источником истины**. Требуется ADR, фиксирующий текущий источник дорожной карты и версию (v3).

## 6. Риски
- Неполная фиксация входов Run и отсутствие PolicySnapshot создают риск несоблюдения требований воспроизводимости и аудита.
- Отсутствие CP↔DP протокола и планировщика блокирует продвижение к M3–M4.
- Отсутствие SSO/RBAC и секретов с TTL блокирует M5 и production‑grade критерии AC‑04/AC‑05.

## 7. Рекомендуемая последовательность устранения
1. **M2 hardening:** схемы/репозитории CodeRef, EnvironmentDefinition/Lock, PolicySnapshot; параметры Run; reproducibility bundle.
2. **M3:** DP runtime + CP↔DP контракт.
3. **M4:** планировщик/очереди/квоты/отмена.
4. **M5:** SSO/RBAC, секреты с TTL, SIEM‑экспорт, retention/legal hold.
5. **M9:** наблюдаемость, HA/DR, Helm+SBOM, автоматизированная E2E‑приёмка.

## 8. Отложенные изменения документации
- README.md: локальные изменения (отложено).
- docs/README_ENTERPRISE_CHECKLIST.md: отложено, файл не отслеживается Git.

Эти изменения будут обработаны в отдельном PR только документации после стабилизации усиления M2 и исполнения M3 в контуре DP.
