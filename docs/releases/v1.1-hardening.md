# v1.1 Hardening — Release Notes

## Scope
- Завершение P5 Integrations Matrix.
- Закрытие P6 Security hardening.
- Закрытие P7 Operational readiness.

## Breaking changes
- Нет.

## Highlights

**Integrations**
- SCM CodeRef verification: политика верификации, фиксация результатов, блокировка при несоответствии.
- Registry integrity: проверка digest и подписи, режимы `allow_unsigned` / `verify_only` / `deny_unsigned`, enforcement на создании EnvLock.
- Webhooks: persistent очередь, детерминированный retry/backoff, idempotency, admin‑инспекция и replay.
- SIEM export: syslog/webhook коннекторы, DLQ и replay, метрики доставки.
- Secrets backend: Vault‑like K8s auth, выдача короткоживущих токенов, доступ к секретам только в DP.

**Security hardening**
- RBAC deny‑by‑default и негативные регрессии.
- Session security: TTL, принудительный logout, ограничение числа сессий.
- Network egress: DP‑policy deny‑by‑default и обязательная сеть‑класс привязка.
- Redaction: запрет утечек секретов в логи/аудит/ответы API.
- Audit: append‑only и экспортируемость без выключения.

**Operational readiness**
- Метрики и SLO‑документация.
- Idempotency и failure‑injection suite для ключевых контуров.
- DR runbooks и env‑gated `dr-validate`.
- Backup/restore сценарии и совместимость бэкапов.
