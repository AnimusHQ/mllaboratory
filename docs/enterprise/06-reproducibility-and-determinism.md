# 06. Воспроизводимость и детерминизм

## 06.1 Определение
Воспроизводимость означает возможность повторного исполнения Run с результатом, согласованным с зафиксированными входами, политиками и окружением, что снижает риск утраты научной ценности.

## 06.2 Обязательные привязки
Воспроизводимый Run фиксирует:
- `DatasetVersion` для всех входных данных;
- `CodeRef` с `commit_sha`;
- `EnvironmentLock` (digest и чеки зависимостей);
- параметры исполнения;
- `PolicySnapshot` и решения;
- требования к ресурсам.

Привязки неизменяемы после создания Run, что снижает риск «дрейфа» условий.

## 06.3 Правила явности
- Образы должны быть digest‑pinned, что снижает риск неявной смены окружения.
- Зависимости и параметры объявляются явно, что снижает риск скрытых дефолтов.
- Зависимости между шагами в Pipeline фиксируются в спецификации, что снижает риск неучтённых связей.

## 06.4 Execution Snapshot и replay
Control Plane сохраняет execution snapshot Run, что снижает риск невозможности воспроизведения.

Replay создаёт новый Run на основании snapshot и фиксируется как отдельная сущность, что снижает риск подмены истории.

## 06.5 Требования production‑Run
Production‑Run допускается только при выполнении условий:
- `CodeRef` содержит `commit_sha`;
- `EnvironmentLock` присутствует и проверяем;
- ссылки на `DatasetVersion` явные и неизменяемые;
- необходимые approvals зафиксированы.

Невыполнение условий приводит к отклонению Run либо к явной фиксации ограничений, что снижает риск недостоверной трактовки результатов.
