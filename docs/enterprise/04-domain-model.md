# 04. Доменная модель

## 04.1 Принципы
Доменная модель фиксирует сущности, их связи и инварианты, что снижает риск неявного состояния и потери воспроизводимости.

Принципы:
1. **Явность** — все материальные элементы представлены сущностями, что снижает риск скрытых зависимостей.
2. **Версионирование** — элементы, влияющие на результат, неизменяемы или имеют историю изменений, что снижает риск «дрейфа».
3. **Контекст проектный** — каждая сущность принадлежит одному проекту, что снижает риск перекрёстного доступа.
4. **Связность** — сущности соединены явными ссылками, что снижает риск ручной реконструкции контекста.
5. **Аудитируемость** — операции порождают `AuditEvent`, что снижает риск спорных трактовок.

## 04.2 Project
Project — корневой контейнер контекста.

**Инварианты:**
- каждая сущность принадлежит ровно одному проекту;
- межпроектные зависимости запрещены.

**Состояния:** `active`, `archived` (read‑only).

## 04.3 Dataset и DatasetVersion
- `Dataset` задаёт логическую группу данных.
- `DatasetVersion` фиксирует неизменяемый снимок данных.

**Инварианты:**
- `DatasetVersion` неизменяема;
- Run ссылается только на конкретную версию;
- удаление возможно только по политике хранения.

## 04.4 CodeRef
`CodeRef` фиксирует источник кода и commit SHA.

**Инварианты:**
- production‑Run использует `commit_sha`;
- ветки/теги не допускаются для production‑Run;
- ссылка неизменяема.

## 04.5 EnvironmentDefinition и EnvironmentLock
- `EnvironmentDefinition` описывает окружение логически.
- `EnvironmentLock` фиксирует исполнимое окружение (digest/чеки) для Run.

**Инварианты:**
- production‑Run использует `EnvironmentLock`;
- `EnvironmentLock` неизменяем;
- проверки digest/подписи обязательны по политике.

## 04.6 Run и PipelineRun
`Run` — единица исполнения и воспроизводимости. `PipelineRun` — DAG из Run.

**Инварианты:**
- Run не изменяет входные данные;
- Run существует только в контексте проекта;
- повторный запуск с теми же входами должен воспроизводиться в пределах модели детерминизма.

## 04.7 Artifact
`Artifact` связан с Run и хранится в объектном хранилище.

**Инварианты:**
- артефакт всегда привязан к Run и проекту;
- доступ контролируется RBAC.

## 04.8 Model и ModelVersion
`ModelVersion` связывает модель с Run, артефактами и контекстом.

**Инварианты:**
- происхождение фиксируется через Run/Artifact;
- экспорт допускается только при выполнении политик.

## 04.9 DevEnv и IDE‑сессии
`DevEnv` предоставляет управляемую IDE‑сессию под теми же правилами governance.

**Инварианты:**
- доступ осуществляется через proxy;
- TTL и аудит обязательны.

## 04.10 PolicySnapshot
`PolicySnapshot` фиксирует применённые правила для Run, что снижает риск «дрейфа» политики.

## 04.11 AuditEvent и Lineage
- `AuditEvent` фиксирует действие (append‑only), что снижает риск подмены истории.
- `Lineage` фиксирует происхождение входов/выходов, что снижает риск утраты provenance.
