# RBAC и принудительное применение доступа

## 1. Назначение и принципиальные правила

RBAC применяется ко всем пользовательским и административным API. По умолчанию действует правило:

- Методы `GET/HEAD/OPTIONS` требуют роль `viewer`.
- Методы `POST/PUT/PATCH/DELETE` требуют роль `editor`.
- Исключения для административных операций задаются явно в сервисе.
- Запросы без проектной области отклоняются с `project_id_required`, если эндпоинт не входит в исключения.
- Отказы в доступе логируются и аудируются как `access.denied` (append‑only).

Run‑token не должен повышать привилегии: административные запросы через run‑token запрещены.

## 2. Поверхности и требуемые роли

### 2.1 Experiments (`/api/experiments/*`)

- Администратор обязателен для: `/policies*`, `/policy-decisions*`, `/policy-approvals*`, `/projects/{project_id}/role-bindings*`, `/projects/{project_id}/webhooks/*`.
- Для остальных эндпоинтов: `viewer` на чтение, `editor` на запись.
- Проектная область обязательна для всех путей, кроме: `/policies*`, `/policy-decisions*`, `/policy-approvals*`, `/model-images*`, `/ci/*`, `/gitlab/*`.

### 2.2 Dataset Registry (`/api/dataset-registry/*`)

- По умолчанию: `viewer` на чтение, `editor` на запись.
- Проектная область обязательна для всех эндпоинтов Dataset/DatasetVersion.

### 2.3 Audit (`/api/audit/*`)

- Все эндпоинты требуют `admin`.
- Административные экспортные операции (`/admin/audit/exports/*`) — `admin`, доступны только через сервис аудита.

### 2.4 Quality (`/api/quality/*`)

- Все эндпоинты требуют `admin`.

### 2.5 Lineage (`/api/lineage/*`)

- Все эндпоинты требуют `admin`.

### 2.6 Gateway (`/api/*`, `/auth/*`)

- Основной слой авторизации для пользовательских токенов.
- Run‑token ограничен набором безопасных путей: `/api/experiments/experiment-runs/{run_id}` (и `/metrics`, `/events`, `/artifacts`, `/stream`), а также `/api/dataset-registry/dataset-versions/{version_id}` и `/download`.
- Административные эндпоинты `/auth/force-logout` и управляемые операции требуют `admin`.

## 3. Внутренние эндпоинты

Внутренние пути `/internal/cp/*` и `/internal/dp/*` обслуживаются только по внутреннему заголовочному токену и не публикуются через Gateway. Они не предназначены для внешнего вызова и защищены отдельно от пользовательских ролей.

## 4. Негативные тесты и регрессии

- Cross‑project доступ запрещён при отсутствии привязок к проекту.
- Понижение роли блокирует операции записи.
- Отсутствие проектной области приводит к детерминированному отказу (`project_id_required`).
